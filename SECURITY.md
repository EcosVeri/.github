# Security Policy

EcosVeri projects are open-source infrastructure. Please report suspected vulnerabilities responsibly.

## Reporting

Do not open a public issue for a suspected vulnerability.

Use private vulnerability reporting when it is available for the affected repository. Otherwise, contact the maintainers privately through GitHub and include:

- Affected repository and version or commit.
- Steps to reproduce.
- Potential impact.
- Any suggested mitigation.

## Response expectations

We aim to acknowledge valid reports within 7 days and provide a remediation plan when the issue is confirmed.

## Scope

Security reports may include dependency vulnerabilities, unsafe defaults, authentication or authorization flaws, data exposure, prompt injection pathways that leak protected data, and reproducibility issues with security impact.
