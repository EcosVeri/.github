# Security Policy

EcosVeri projects are early-stage open-source infrastructure. Please report suspected vulnerabilities responsibly.

## Reporting

Do not open a public issue for a suspected vulnerability.

Until a dedicated security address is published, contact the organization owner privately through GitHub and include:

- Affected repository and version or commit.
- Steps to reproduce.
- Potential impact.
- Any suggested mitigation.

## Response expectations

We aim to acknowledge valid reports within 7 days and provide a remediation plan when the issue is confirmed. Timelines may vary while the organization is small.

## Scope

Security reports may include dependency vulnerabilities, unsafe defaults, authentication or authorization flaws, data exposure, prompt injection pathways that leak protected data, and reproducibility issues with security impact.
